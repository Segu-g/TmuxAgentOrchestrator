## Role Focus

You are a **Worker** agent. Your scope is execution, not coordination:
- Implement one task at a time, following TDD (Red → Green → Refactor)
- Do NOT spawn sub-agents unless the task explicitly requires parallelisation
- Do NOT submit tasks via REST — use `/task-complete` to signal completion
- Stay within your worktree; do not read or write files in other agents' directories

## Task Completion Protocol

Task completion is **always explicit**. The orchestrator never auto-detects it.

When ALL work for your current task is done:

```
/task-complete <one-line summary of what was accomplished>
```

Rules for calling `/task-complete`:
- Call it **exactly once** per task
- Only call it after all files are committed and tests pass
- Do NOT call it mid-task or speculatively
- Do NOT call it if there are failing tests or uncommitted changes

After each Claude response turn, the Stop hook checks whether you have called `/task-complete`.
If you have not, you receive a nudge. Respond to nudges by either continuing work or calling
`/task-complete` if you are genuinely done.

## TDD Workflow

Follow the Red → Green → Refactor cycle for all implementation:

1. **Plan** — write `PLAN.md` with acceptance criteria using `/plan <description>`
2. **Red** — write a failing test that captures the requirement
3. **Green** — write the minimal code to make the test pass
4. **Refactor** — improve code quality without changing behaviour
5. **Commit** — `git add -A && git commit -m "feat: <description>"`
6. **Complete** — call `/task-complete <summary>`

Never commit failing tests. Never skip the planning step for non-trivial work.

## Context Management for Workers

Recommended sequence: **Write → Select → Compress → Isolate**

- **Write**: Save progress to `NOTES.md` and `PLAN.md` before context fills up
- **Select**: At each new task, re-read `NOTES.md` and relevant source files only
- **Compress**: Run `/summarize` when your context exceeds ~60% — do not wait until it is full
- **Isolate**: Use `/spawn-subagent` only if a sub-task is truly independent and parallelisable

Signs of context rot: repeating yourself, forgetting earlier decisions, missing files you already
created. Run `/summarize` immediately if you notice these symptoms.

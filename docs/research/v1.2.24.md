### Step 1 — 調査記録 (2026-03-15)

#### Query 1: peer review multi-agent LLM parallel implementation review 2025

**Source**: AgentReview: Exploring Peer Review Dynamics with LLM Agents (EMNLP 2024)
https://arxiv.org/abs/2406.12708

**Source**: GitHub - Ahren09/AgentReview
https://github.com/Ahren09/AgentReview

**Key findings**:
- AgentReview は reviewers・authors・Area Chairs を LLM エージェントとしてシミュレートする最初のフレームワーク
- 5フェーズパイプライン: 初期レビュー → 著者リバッタル → レビュアー・ACディスカッション → メタレビュー → 最終判定
- レビュアーのバイアスにより37.1%の決定変動が観測 → 複数レビュアーによる独立評価の重要性を実証
- 並列独立レビューはバイアスを低減し判断精度を向上させる

#### Query 2: code review workflow multi-agent software development pipeline best practices 2025

**Source**: "Scaling Code Review: Multi-Agent Systems for Enterprise Engineering Teams"
https://rkoots.github.io/blog/2026/03/09/bringing-code-review-to-claude-code/ (2026-03)

**Source**: arXiv:2505.16339 "Rethinking Code Review Workflows with LLM Assistance"
https://arxiv.org/html/2505.16339v1 (2025)

**Key findings**:
- 4専門エージェント (Security/Performance/Architecture/DomainLogic) が並列で同一PRを分析する「adversarial validation」パターン
- 40% faster merge cycles; 84% of large PRs with critical issues found
- 各エージェントの知見を cross-challenge (相互否定) することで false positive < 1%
- 動的リソース割り当て: 複雑な変更 → full agent team; マイナー修正 → lightweight analysis

#### Query 3: AgentReview academic peer review LLM simulation parallel agent 2024

**Source**: AgentReview: Exploring Peer Review Dynamics with LLM Agents — EMNLP 2024
https://aclanthology.org/2024.emnlp-main.70/

**Key findings**:
- 53,800以上のレビュー・リバッタル・メタレビューを生成したシミュレーション実績
- 社会影響理論・altruism fatigue・authority bias が peer review バイアスの主因
- 独立レビュアーの並列評価 + AC (Area Chair) による統合判定が最高品質の判断を生む
- これをコードレビューに適用: impl-A∥impl-B 並列実装 + reviewer による比較選択

### Design Decisions

1. **`PeerReviewWorkflowSubmit`**: `feature`, `language`, `impl_a_tags`, `impl_b_tags`, `reviewer_tags`, `reply_to` フィールド
2. **DAG 構造**: `impl-a` と `impl-b` が並列 (no depends_on) → `reviewer` が両者に depends_on
3. **スクラッチパッドキー**: `{prefix}_impl_a`, `{prefix}_impl_b`, `{prefix}_review`
4. **レビュアープロンプト**: 両実装を読み込み、可読性・正確性・エッジケース処理・コードスタイルの4軸で比較。REVIEW.md + winner 宣言
5. **agent_plugin/docs/README.md**: ロール一覧表・フォーマット規約・ワークフロー対応表・新規追加手順

# v1.1.5 — web/app.py APIRouter 分割 + `get_agent_dict` O(1) 最適化

## 問題

`web/app.py` は v1.1.4 時点で **7434 行**と肥大化しており、以下の問題がある:

1. **認知的負荷**: エンドポイントを追加・修正するたびに 7000+ 行のファイルをナビゲートする必要がある
2. **マージコンフリクト**: 複数の変更が同一ファイルに集中する
3. **テスト性の低下**: 全ルートが `create_app()` クロージャ内にネストされており、個別ルーターのユニットテストが困難
4. **`get_agent_dict` O(n)**: `Orchestrator.get_agent_dict()` が `list_all()` で全エージェントを構築してから線形探索する

## 解決策

### 1. `web/schemas.py` — Pydantic スキーマ分離

`app.py` の冒頭 ~1050 行にある Pydantic モデル群を `web/schemas.py` に移動する。
`app.py` は `from tmux_orchestrator.web.schemas import *` で後方互換性を保つ。

### 2. `web/routers/` — ドメイン別 APIRouter

builder 関数パターン: 各モジュールが `def build_<domain>_router(orchestrator, auth, ...) -> APIRouter` を export する。

ドメイン分割:
- `web/routers/agents.py` — /agents/* エンドポイント
- `web/routers/tasks.py` — /tasks/* エンドポイント
- `web/routers/workflows.py` — /workflows/* エンドポイント
- `web/routers/scratchpad.py` — /scratchpad/* エンドポイント
- `web/routers/system.py` — /health, /metrics, /dlq, /audit-log, /checkpoint, /telemetry, /drift, /results
- `web/routers/webhooks.py` — /webhooks/* エンドポイント
- `web/routers/groups.py` — /groups/* エンドポイント

`create_app()` は各 builder を呼んで `app.include_router()` するだけになる。

### 3. `registry.py` — `get_one_dict()` O(1) メソッド追加

```python
def get_one_dict(self, agent_id: str, drop_counts: dict[str, int] | None = None) -> dict | None:
    agent = self._agents.get(agent_id)
    if agent is None:
        return None
    drops = drop_counts or {}
    a = agent
    return {
        "id": a.id,
        "status": a.status.value,
        ...  # same shape as list_all()
    }
```

`Orchestrator.get_agent_dict()` は `list_all()` の代わりに `registry.get_one_dict()` を呼ぶ。

## レビュー指摘

**[指摘1]** `web/schemas.py` のバリデーター — 旧 `app.py` では Pydantic `field_validator` が `from pydantic import field_validator` としてクラスボディ内でインポートされていた（非標準だが Python では有効）。`schemas.py` ではモジュールレベルの `from pydantic import BaseModel, field_validator, model_validator` に修正済み。検証済み: 全バリデーターが正しく動作する。

> 対応: モジュールレベルインポートに整理済み。`uv run python -c "from tmux_orchestrator.web.schemas import ..."` で全バリデーターを手動確認済み。

**[指摘2]** `get_one_dict()` の field shape — `list_all()` と完全に同一の dict 構造であることをテストで担保すること。

> 対応: `test_get_one_dict_matches_list_all_field_shape()` でキーセットの完全一致と主要フィールドの値一致を検証済み。

**[指摘3]** `app.py` からの後方互換再エクスポート — `from tmux_orchestrator.web.app import TaskSubmit` が引き続き動作することをテストで担保すること。

> 対応: `test_schemas_re_exported_from_app()` で確認済み。

**[指摘4]** `app.py` 内の `from pydantic import BaseModel` が不要になった点 — schema クラスをすべて外部モジュールに移したため、app.py 内で BaseModel を直接参照する箇所がなくなった。削除済み。

> 対応: 削除済み (`grep -n "BaseModel" app.py` で import 行のみ → 削除後ゼロ件確認済み)。

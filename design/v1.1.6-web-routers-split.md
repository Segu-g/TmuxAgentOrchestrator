# v1.1.6 — web/routers/ APIRouter 実装（エンドポイントハンドラの物理分割）

## 問題

`web/app.py` は v1.1.5 時点で **6463 行**。v1.1.5 でスキーマを `web/schemas.py` に分離したが、
エンドポイントハンドラは依然として `create_app()` クロージャ内に全て集積している。

問題点:
1. **認知的負荷**: 新エンドポイント追加・修正の際に 6000+ 行をナビゲート
2. **マージコンフリクト**: 複数変更が同一ファイルに集中
3. **テスト性の低下**: 全ルートが `create_app()` クロージャ内にネストされており個別テストが困難
4. **スコープ過大**: エージェント管理・タスク管理・ワークフロー・スクラッチパッド・システムが混在

## 解決策

### builder 関数パターン

各ルーターファイルが `def build_<domain>_router(orchestrator, auth, ...) -> APIRouter` を export する。
`create_app()` はこれらを呼び出して `app.include_router()` するだけになる。

### 分割ドメイン

| ファイル | prefix | 主なエンドポイント |
|---|---|---|
| `web/routers/agents.py` | /agents | GET /agents, POST /agents, DELETE /agents/{id}, POST /agents/{id}/reset, /task-complete, /ready, /drain, /message, etc. |
| `web/routers/tasks.py` | /tasks | GET /tasks, POST /tasks, POST /tasks/batch, PATCH /tasks/{id}/priority, DELETE /tasks/{id} |
| `web/routers/workflows.py` | /workflows | POST /workflows, POST /workflows/tdd, /debate, /adr, /delphi, /redblue, /socratic, /pair, /fulldev, /clean-arch, /ddd, /competition; GET /workflows/* |
| `web/routers/scratchpad.py` | /scratchpad | GET /scratchpad/, PUT /scratchpad/{key}, GET /scratchpad/{key}, DELETE /scratchpad/{key} |
| `web/routers/system.py` | — | /health*, /metrics, /dlq, /audit-log, /checkpoint/*, /telemetry/status, /drift, /results/*, /orchestrator/status, /pause-dispatch, /resume-dispatch, /rate-limit, /autoscaler |
| `web/routers/webhooks.py` | /webhooks | POST /webhooks, GET /webhooks, DELETE /webhooks/{id}, GET /webhooks/{id}/deliveries |
| `web/routers/groups.py` | /groups | POST /groups, GET /groups, GET /groups/{name}, DELETE /groups/{name}, POST/DELETE /groups/{name}/agents/{id} |
| `web/routers/memory.py` | /agents | GET/POST/DELETE /agents/{id}/memory/* (エピソード記憶) |

### `web/routers/__init__.py`

空ファイル（Python パッケージ化）

### `app.py` 縮小

- `create_app()` から全ハンドラを削除
- 各 `build_*_router()` を呼び出す `include_router()` 群に置換
- 認証 (`auth`)・limiter・状態 (`_scratchpad` 等) は `app.py` で作成して各 router に渡す

## テスト計画

- 全既存テスト (1907+) が引き続き green であること (回帰)
- `test_web_routers.py` — 各ルーター builder が `APIRouter` インスタンスを返すことを確認
- `test_router_endpoints.py` — 各ルーターに期待されるエンドポイントが存在することを確認
- OpenAPI スナップショット再生成: `UPDATE_SNAPSHOTS=1 uv run pytest tests/test_openapi_schema.py`

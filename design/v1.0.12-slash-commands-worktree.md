# v1.0.12 — スラッシュコマンド群をエージェントワークツリーで使用可能にする

## 問題の背景

v1.0.11 デモで 14/15 のチェックがパスした。失敗した 1 件は `agent-verifier` が作業完了後に `/task-complete` を呼び出さなかった問題。

### 根本原因

スラッシュコマンドは `agent_plugin/commands/` に定義されており、`claude` CLI の起動時に `--plugin-dir` フラグで読み込まれる:

```bash
claude --dangerously-skip-permissions --plugin-dir /path/to/agent_plugin
```

このとき、コマンドは名前空間付きの `/tmux-orchestrator:task-complete` という形式でのみ利用可能になる。エージェントが記述する CLAUDE.md にも `/tmux-orchestrator:task-complete` と表記されている。

しかし、実際のエージェントが作業中に使用しようとするとき、以下の問題が発生する:

1. **名前空間付きコマンドの認識ミス**: エージェントは `/task-complete` を試みるが、`/tmux-orchestrator:task-complete` が正しい形式
2. **ワークツリーのコマンド非存在**: ワークツリー内の `.claude/commands/` にコマンドファイルが存在しないため、シンプルな `/task-complete` では動作しない
3. **文書との乖離**: CLAUDE.md のスラッシュコマンドリファレンスで名前空間付き形式を案内しているが、エージェントは直感的に `/task-complete` を使う傾向がある

## 提案する解決策

エージェント起動時（ワークツリーセットアップ後、`claude` プロセス起動前）に、`agent_plugin/commands/` のすべての `.md` ファイルを各エージェントの `{worktree}/.claude/commands/` にコピーする。

これにより:
- エージェントは `/task-complete`、`/check-inbox` 等をプレーン形式で使用できる
- `--plugin-dir` 経由の名前空間付き形式も引き続き動作（破壊的変更なし）
- CLAUDE.md の更新（名前空間なし形式に統一）でエージェントの理解を助ける

### 対象エージェント

| エージェント種別 | 対象の cwd | 動作 |
|---|---|---|
| `isolate=True` (デフォルト) | ワークツリーパス | `.claude/commands/` を作成してコマンドをコピー |
| `isolate=False` | `cwd_override` または `Path.cwd()` | 同様に `.claude/commands/` にコピー（ただし既存ファイルは上書きしない） |

## 実装計画

### 1. `_copy_commands()` メソッド追加 (`claude_code.py`)

```python
def _copy_commands(self, cwd: Path) -> None:
    """Copy agent_plugin/commands/ → {cwd}/.claude/commands/

    Makes slash commands available as /task-complete etc. (without namespace prefix).
    Does not overwrite files that already exist in the destination.
    """
    commands_src = Path(__file__).parent.parent / "agent_plugin" / "commands"
    if not commands_src.is_dir():
        return
    commands_dst = cwd / ".claude" / "commands"
    commands_dst.mkdir(parents=True, exist_ok=True)
    for cmd_file in commands_src.glob("*.md"):
        dest = commands_dst / cmd_file.name
        if not dest.exists():  # don't overwrite existing customizations
            shutil.copy2(cmd_file, dest)
            logger.debug("Agent %s: copied command %s → %s", self.id, cmd_file.name, dest)
```

### 2. `start()` への組み込み

既存の `_copy_context_files()` および `_copy_context_spec_files()` の直後に呼び出す:

```python
await loop.run_in_executor(None, self._copy_commands, cwd)
```

### 3. CLAUDE.md のスラッシュコマンドリファレンス更新

名前空間付き形式 (`/tmux-orchestrator:task-complete`) から名前空間なし形式 (`/task-complete`) に変更する。

## レビュー指摘

### 指摘 1: `cwd is None` の場合のハンドリング

`start()` の `if cwd is not None:` ブロック内で `_copy_commands()` を呼び出すため、`cwd=None` の場合はコピーがスキップされる。これは既存の `_copy_context_files()` と一貫した挙動であり正しい。ただし、設計文書で明示しておく必要がある。

> 対応: `cwd is not None` の場合のみコピーすることをコード内のコメントに明記する。

### 指摘 2: `isolate=False` のエージェントで `.claude/commands/` への書き込みが適切か

`isolate=False` のエージェントは共有 cwd（プロジェクトルート等）で動作する。`.claude/commands/` を書き込むことで、他のエージェントや既存のプロジェクト設定を破壊する可能性がある。

> 対応: `isolate=False` のエージェントに対しても `exist_ok=True` で作成し、既存ファイルは上書きしない方針を維持する。これは上書きしないロジック（`if not dest.exists()`）で対処済み。共有 cwd の `.claude/commands/` に書き込むことはリスクが低い（Claude の設定フォルダは通常プロジェクトに含まれない）。

### 指摘 3: CLAUDE.md 内のスラッシュコマンドリファレンスの更新対象

CLAUDE.md には Worker ロールと Director ロールそれぞれにコマンドリファレンスのテーブルが含まれている。名前空間付き形式 (`/tmux-orchestrator:task-complete`) から名前空間なし形式 (`/task-complete`) への変更は、コマンドのコピーと一緒に行う必要がある。また、CLAUDE.md 上部の「Task Completion」セクションも更新が必要。

> 対応: `_write_agent_claude_md()` 内のスラッシュコマンドリファレンステーブルと Task Completion セクションの両方を更新する。

### 指摘 4: サブエージェント（`parent_pane` あり）の場合のワークツリーパス

`parent_pane` が設定されているサブエージェントの場合、worktree_manager の有無によって `cwd` の決定方法が異なる。`cwd` が None にならないよう注意が必要。

> 対応: サブエージェントも通常の `_setup_worktree()` を通るため、`cwd` が正常に解決される。特別な対応は不要。

### 指摘 5: テスト内で実際の `agent_plugin/commands/` ディレクトリに依存すること

テストが実際のファイルシステムパス（`Path(__file__).parent.parent / "agent_plugin" / "commands"`）に依存する場合、テスト実行環境によって結果が変わる可能性がある。

> 対応: テストでは `tmp_path` に模擬の `agent_plugin/commands/` ディレクトリを作成してテストする。または、`_copy_commands()` に `commands_src` を注入できるようにする。実際のテストでは `patch` を使って `agent_plugin/commands/` の存在を制御する方法が最も堅牢。ただし単純に `tmp_path` にフェイクのコマンドディレクトリを作るだけでもよい。

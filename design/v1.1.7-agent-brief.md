# v1.1.7 — POST /agents/{id}/brief: Out-of-Band Context Injection

## Problem

エージェントが長時間タスクを実行中に、要件変更や追加情報を注入する手段がなかった。
エージェントを中断せずに新しいコンテキストを伝える仕組みが必要。

## Solution

`POST /agents/{agent_id}/brief` エンドポイントを追加する。

### 配信メカニズム

1. `brief_id` (UUID) を生成（呼び出し元が指定可能）
2. エージェントのワークツリー `worktree_path/__brief__/{brief_id}.txt` にコンテンツを書き込む
3. `agent.notify_stdin(f"__BRIEF__:{brief_id}")` でtmux paneに通知
4. エージェントは `/read-brief {brief_id}` コマンドでコンテンツを取得

### Stop Hook との競合回避

`notify_stdin()` は Claude Code の Stop Hook を一切トリガーしない。
（Stop Hook は Claude の応答ターン完了時のみ発火する）
これにより §10.40/§10.41 で記録された「複雑」な懸念が完全に解消される。

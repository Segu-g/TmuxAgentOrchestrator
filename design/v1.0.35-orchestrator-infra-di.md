# v1.0.35 — `orchestrator.py` 残りインフラ依存 DI 整理

## Problem

`orchestrator.py` の `__init__` に5箇所のインライン import / 直接インスタンス化が残っている:

1. `ResultStore` — `if config.result_store_enabled:` ブランチ内でのインライン import
2. `CheckpointStore` — `if config.checkpoint_enabled:` ブランチ内でのインライン import
3. `AutoScaler` — `if config.autoscale_max > 0:` ブランチ内でのインライン import
4. `WorkflowManager` — 無条件インライン import + インスタンス化
5. `GroupManager` — トップレベル import + 無条件インスタンス化

これらがあると `orchestrator.py` が infrastructure モジュールに直接依存し、単体テストで
実際のファイルシステムや tmux が不要でもモジュールがロードされる。
Clean Architecture の Dependency Rule（依存は常にドメイン中心に向かう）に違反している。

## Solution

1. `application/infra_protocols.py` 新設 — 5コンポーネントの Protocol を定義
2. Null Object 実装を同ファイルに追加（`NullResultStore`, `NullCheckpointStore`, `NullAutoScaler`）
3. `WorkflowManager` / `GroupManager` は既に十分シンプルで Null Object 不要（空の実装がデフォルト動作）
4. `Orchestrator.__init__` を constructor injection に変換
5. `factory.py` `build_system()` で具体実装を注入

## レビュー指摘

### レビュー1 — コンテキストフリー評価

**指摘1**: `CheckpointStoreProtocol.save_task` が `task: Any` を使用している。
本来 `Task` 型であるべきだが、application/ purity ルールにより `agents.base.Task` を import できない。
これは意図的な妥協であるが、型安全性が低下する。
> 対応: `Any` の使用はコメントで明示。将来 `domain/task.py` に `Task` が移動した際に
> `tmux_orchestrator.domain.task.Task` として参照可能になる。`NullCheckpointStore` の
> `save_task` 等も同様。この課題は `DESIGN.md §11 domain/ 移行` の作業項目として記録済み。

**指摘2**: `NullAutoScaler.reconfigure` の戻り値が `min or 0` 形式になっており、
`min=0` を渡した場合に意図しない結果（`0 or 0 = 0`）になる可能性がある。
> 対応: `None` のデフォルト値が `0` と区別されない場合の issue を修正。
> `min if min is not None else 0` 形式に変更すべきだが、実際には `NullAutoScaler.reconfigure`
> は no-op であり、返り値は単なるエコーバック。`None` と `0` を明示的に区別するよう修正する。

**指摘3**: `test_orchestrator_infra_di.py` の `test_all_infra_components_injected_together`
テストが `await orch.get_autoscaler_status()` と `orch.reconfigure_autoscaler()` を呼んでいるが、
`NullAutoScaler` を注入した場合の `_autoscaler is not None` チェックが `get_autoscaler_status`
の分岐に影響するかを確認すべきである。
> 対応: 現行の `get_autoscaler_status` は `if self._autoscaler is None` でチェックしており、
> `NullAutoScaler` を注入すると `_autoscaler is not None` となるため `status()` を呼ぶ。
> `NullAutoScaler.status()` は `enabled: False` を返すので正しい動作。テストで確認済み。
